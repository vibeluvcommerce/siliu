# Siliu Browser 项目规划文档

> 本文档整理项目发展方向、商业模式和技术架构规划

---

## 一、项目定位

**Siliu Browser** 是一款 **AI 驱动的个人自动化浏览器**，核心目标是让非技术用户通过自然语言指令完成复杂的网页操作。

### 核心价值
1. **自然语言控制** - "帮我抓取这个页面的所有商品信息"
2. **Agent 生态** - 针对不同网站的专门优化（B站、淘宝等）
3. **可视化生成** - 无需编程创建自动化流程

### 目标用户
- **核心用户**：技术爱好者、效率工具用户、数据分析师
- **扩展用户**：电商运营、内容创作者、研究人员
- **企业用户**：中小企业自动化需求

## 二、技术架构规划

### 2.1 当前架构

```
Siliu Browser
├── Core (Electron + CDP/JS 双模式)
├── Copilot (AI 决策引擎)
├── Agent System (网站专用适配器)
│   ├── builtin/ (官方 Agent)
│   └── custom/ (用户自定义)
└── Export Center (数据导出)
```

### 2.2 配置化 Agent 架构（规划中）

**问题**：普通用户无法编写 JS 代码创建 Agent

**解决方案**：YAML 配置 + 可视化编辑器

```
~/.siliu/workspace/agents/
├── my-bilibili.yaml      # YAML 配置化 Agent
├── my-taobao.yaml
└── ...

src/copilot/agents/
├── configurable-agent.js     # 配置化 Agent 基类
├── dynamic-agent-loader.js   # 动态加载器
└── builtin/                  # 内置 Agent（保持不变）
```

### 2.3 坐标优先级策略

| 优先级 | 方法 | 适用场景 |
|--------|------|----------|
| **1** | 坐标 (x,y) | 固定布局（导航栏、按钮）|
| **2** | JS Selector | 动态内容（列表、评论区）|
| **3** | 混合策略 | 复杂交互场景 |

**Agent 预置坐标示例**：
```yaml
coordinates:
  searchBox:
    x: 0.52
    y: 0.06
    description: 顶部搜索框
    action: click-and-type
```

### 3.2 Agent 市场机制

- **官方 Agent**：免费提供给 Pro 用户
- **社区 Agent**：创作者上传，平台抽成 20-30%
- **企业 Agent**：独立定价

## 五、风险评估

| 风险 | 等级 | 缓解策略 |
|------|------|----------|
| 网站改版导致坐标失效 | 高 | 快速更新机制 + 社区贡献 |
| 法律合规 | 中 | 明确用户协议 + 频率限制 |
| AI 准确率 | 中 | 多模态增强 + 人工校验 |
| 竞品竞争 | 中 | 差异化定位（开源 + 个人用户）|

## 六、关键决策

1. **Agent 生成器**：先做 Web 表单版，再做可视化标注
2. **订阅系统**：使用现成方案（Lemonsqueezy/Paddle）
3. **开源策略**：核心功能开源，高级功能订阅
