# Siliu 测试计划文档

## 📋 项目概述

**Siliu** 是一款 AI 驱动的浏览器自动化工具，核心功能是通过自然语言指令控制浏览器完成复杂网页操作。

### 核心特性
- 🤖 AI Copilot：支持 Kimi、MiniMax 等云端 AI 服务
- 🎯 视觉驱动：通过截图理解页面状态，坐标点击精准操作
- 🔄 CDP 控制：基于 Chrome DevTools Protocol 的浏览器控制
- 📊 数据导出：支持 Excel、CSV、JSON、PDF 导出
- 🤖 Agent 系统：可配置的专业领域自动化助手

## 🧪 详细测试用例

### 测试用例 1: 基础导航

**目的**: 验证浏览器基本导航功能

**步骤**:
1. 启动 Siliu
2. 在地址栏输入 `https://www.google.com`
3. 按 Enter 导航
4. 等待页面加载完成
5. 点击链接进入其他页面
6. 点击后退按钮
7. 点击刷新按钮

**预期结果**:
- 页面正常加载
- 前进/后退/刷新功能正常
- 地址栏 URL 正确更新

---

### 测试用例 2: AI Copilot 基础对话

**目的**: 验证 AI Copilot 基础功能

**前置条件**:
- 已配置 AI 服务（Kimi 或 MiniMax）

**步骤**:

**预期结果**:
- AI 正确理解指令

## 📊 测试环境

### 推荐测试环境

**Windows**:
- Windows 10/11 64位
- Node.js 18+
- Electron 26.6.10

**硬件要求**:
- CPU: Intel i5 或更高
- RAM: 8GB 或更高
- 网络: 稳定的互联网连接（AI 服务需要）

### AI 服务配置

**Kimi**:
- API Endpoint: https://api.moonshot.cn/v1
- API Key: 从 Moonshot 获取
- 模型: kimi-k2.5 或 k2p5

**MiniMax**:
- API Endpoint: https://api.minimax.chat/v1
- API Key: 从 MiniMax 获取
- 模型: MiniMax-M2.5-highspeed

## 🐛 常见问题排查

### 问题 1: AI 服务连接失败

**症状**: AI Copilot 显示"连接失败"

**排查步骤**:
1. 检查网络连接
2. 检查 API Key 是否正确
3. 检查 API Endpoint 是否可访问
4. 查看控制台错误日志

### 问题 2: 坐标点击不准确

**症状**: AI 点击位置与预期不符

**排查步骤**:
1. 检查视口尺寸是否匹配
2. 检查 DPR（设备像素比）
3. 确认截图和点击使用相同的视口信息
4. 检查页面是否有缩放

### 问题 3: 页面加载超时

**症状**: 导航后页面长时间不加载

**排查步骤**:
1. 检查网络连接
2. 检查 URL 是否正确
3. 检查是否有弹窗阻挡
4. 查看控制台网络日志
