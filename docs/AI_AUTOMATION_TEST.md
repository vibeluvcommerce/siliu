# Siliu AI 自动化能力测试方案

> **版本**: v1.0.0  
> **适用**: Siliu Browser v2.2.0+  
> **测试目标**: 验证 AI 能否自主完成真实场景任务

---

## 测试理念

**不测功能，测任务完成度**

不关注"点击是否正常"，而关注"AI 能否自主完成上传视频、保存图片、采集数据等真实任务"。

---

## 一、核心测试维度

| 维度 | 说明 | 关键问题 |
|------|------|----------|
| **🎯 任务理解** | AI 能否正确理解用户意图 | 会不会走错流程？ |
| **🔧 工具选择** | AI 能否选对操作 | 该用 click 还是 saveImage？ |
| **🧭 自主导航** | AI 能否自主调整策略 | 页面变了怎么办？ |
| **📊 结果反馈** | AI 能否报告执行结果 | 文件保存到哪了？ |

---

## 二、必测场景（P0 - 核心能力）

### 场景 1: 保存网页图片（saveImage）

**用户指令**: 
```
打开 Unsplash 首页 https://unsplash.com，等待页面加载完成后，找到页面上的第一张图片，使用 saveImage 操作在图片位置触发下载，然后使用 download 操作完成保存，最后告诉我图片的文件名、大小和保存路径。
```

**测试重点**:
- ✅ AI 知道先使用 saveImage，再用 download
- ✅ AI 能从截图中识别图片位置
- ✅ 蓝色标记 `(0.22, 0.81)` 显示在正确位置
- ✅ AI 能收到文件信息并报告给用户

**通过标准**:
- [ ] 任务在 5 步内完成
- [ ] AI 正确报告：文件名、大小（如 282KB）、完整路径
- [ ] 图片可正常打开

## 六、测试执行方式

### 方式 1: 完整流程测试（推荐）

直接复制「用户指令」给 Copilot，观察 AI 自主执行：

```
期望行为:
1. AI 分析任务目标
2. 自主规划步骤
3. 执行并确认每步成功
4. 最终报告结果
```

### 方式 2: 分步引导测试（调试）

当完整流程失败时，手动引导定位问题：

```
用户: "打开 Unsplash"
AI: [执行 navigate]
用户: "截图看看"
AI: [执行 screenshot]
用户: "第一张图片在哪"
AI: [分析并给出坐标]
用户: "用 saveImage 保存"
AI: [执行 saveImage]
...
```

### 方式 3: 对比测试

同一任务对比不同 Agent：

```
通用助手 vs B站助手:
- 同样的"悬停头像"任务
- 观察坐标选择差异
- 验证 Agent 特有知识是否生效
```

|------|------|------|------|
| **P0 - 阻塞** | 核心功能完全不可用 | saveImage 崩溃、下载无响应 | 立即修复 |

### 场景 N: 右键保存图片（saveImage）

#### 目标
测试 saveImage 操作 - 通过坐标直接下载嵌入式图片（复用页面 Cookie，支持防盗链图片）

#### 前置条件
- 使用 Unsplash 或其他有嵌入式图片的网站

#### 完整流程（直接复制给 Copilot）

```
打开 Unsplash 首页 https://unsplash.com，等待页面加载完成后，找到页面上的第一张图片，使用 saveImage 操作在图片位置触发下载，然后使用 download 操作完成保存，最后告诉我图片的文件名、大小和保存路径。
```

#### 分步操作

| 步骤 | AI 指令 | 预期结果 |
|------|---------|----------|
| 1 | "打开 https://unsplash.com" | 页面加载，显示图片网格 |
| 2 | "找到第一张图片的坐标位置" | AI 识别图片位置 |
| 3 | "使用 saveImage 操作触发图片下载" | 在图片坐标处显示蓝色标记 `(0.22, 0.81)` 并触发下载 |
| 4 | "使用 download 操作完成保存" | 系统对话框被拦截，文件保存到工作区 |
| 5 | "告诉我图片保存信息" | AI 报告文件名、大小和完整路径 |

#### AI 操作示例

```json
// 第1步：触发图片下载（显示蓝色坐标标记）
{"action": "saveImage", "target": {"type": "coordinate", "x": 0.22, "y": 0.81}, "savePath": "~/.siliu/workspace/downloads/", "description": "在图片位置触发下载"}

// 第2步：完成下载（处理系统对话框）
{"action": "download", "downloadPath": "~/.siliu/workspace/downloads/", "description": "完成图片保存"}
```

#### 工作流程说明

**两步流程设计**：
1. **saveImage**: 在指定坐标处获取图片 URL → 显示蓝色标记 `(x, y)` → 触发下载 → 系统对话框弹出
2. **download**: 拦截系统对话框 → 自动填充路径 → 确认保存 → 监控下载完成

**技术特点**：
- **复用页面 Cookie**: 使用 `webContents.downloadURL()` 下载，自动携带页面 Session，支持防盗链图片
- **高分辨率**: 自动获取 `srcset` 中的最大尺寸图片
- **蓝色坐标标记**: 标记显示相对坐标如 `(0.22, 0.81)`，方便调试
- **强制工作区**: 所有文件最终保存在 `~/.siliu/workspace/downloads/`
- **自动防重复**: 文件名冲突时自动添加序号 `image(1).jpg`

#### 验证方法

执行完成后，AI 应收到完整信息：
```
【上一步结果】
执行成功 ✓
文件 "unsplash_image.jpg" (282.18 KB) 已下载完成，保存路径: C:\Users\...\.siliu\workspace\downloads\unsplash_image.jpg
文件名: unsplash_image.jpg
文件大小: 282.18 KB
保存路径: C:\Users\...\.siliu\workspace\downloads\unsplash_image.jpg
```

#### 通过标准
- [ ] 页面正确加载并显示图片
- [ ] 蓝色坐标标记正确显示在图片位置
- [ ] saveImage 成功触发下载
- [ ] download 操作成功拦截对话框
- [ ] 图片成功保存到工作区
- [ ] AI 收到包含文件名、大小的完整信息
- [ ] 图片能正常打开

#### 适用场景
- 知乎文章配图（无下载链接，有防盗链）
- 微信公众号图片
- Unsplash/Pexels 等图片网站
- 任何 `<img>` 标签显示的图片

#### 故障排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 标记显示但下载未触发 | 坐标位置无图片元素 | 调整坐标后重试 |
| 对话框未弹出 | 网络延迟 | 增加 wait 时间 |
| 下载失败 | 图片有防盗链 | saveImage 已复用 Cookie，检查网络连接 |
