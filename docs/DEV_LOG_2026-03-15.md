## 1. 工作区目录统一管理 ✅ 已完成

### 问题背景
项目原位于 OneDrive 同步文件夹，导致：
- 窗口创建延迟 200-500ms
- 截图、临时文件被不必要地同步
- 文件 I/O 性能下降

### 解决方案
创建统一工作区 `~/.siliu/workspace/`（非 OneDrive 目录）

### 目录结构
```
~/.siliu/workspace/
├── screenshots/          # 截图（从 os.tmpdir() 迁移）
├── auto-files/
│   ├── uploads/         # AI 上传文件
│   └── downloads/       # AI 下载文件
├── exports/             # 导出文件（PDF、图片、数据等）
├── tasks/               # 任务数据（内部使用）
├── cache/               # 临时缓存（内部使用）
├── downloads/           # 浏览器下载（内部使用）
└── .gitignore
```

## 2. 数据导出功能 ⚠️ 基础实现完成，需进一步测试

### 功能概述
实现统一数据导出能力，支持 AI 分步采集、自动合并、多格式导出。

### 核心设计
```
AI collect action → 磁盘缓存 → 合并 → 格式转换 → 导出文件
```

### 结束判断机制
- **正常结束**：AI 输出 `hasMore: false`
- **超时结束**：180秒无新数据自动导出
- **状态提示**：
  - 完整导出："✅ 已完整导出数据"
  - 超时导出："⚠️ 已导出部分数据（超时）"

### 支持格式
| 格式 | 图片处理 | 依赖 |
|------|---------|------|
| Excel (.xlsx) | 嵌入单元格 | exceljs |
| CSV (.csv) | URL 文本 | csv-writer |
| JSON (.json) | 保留 URL | 原生 |
| PDF (.pdf) | 嵌入文档 | puppeteer |
| PNG (.png) | 图表生成 | puppeteer |

### AI 使用方式
```javascript
// 分批采集数据
{
  action: 'collect',
  content: {
    type: 'table',
    data: {
      headers: ['商品', '价格', '图片'],
      rows: [
        ['iPhone', 5999, { type: 'image', url: 'https://...', alt: 'iPhone' }]
      ]
    }
  },
  batchIndex: 0,
  hasMore: true  // 还有下一页
}

// 最后一页
{
  action: 'collect',
  // ...
  batchIndex: 5,
  hasMore: false  // 触发导出
}
```
