# Siliu 浏览器自动化测试场景手册

> **版本**: v2.1.0  
> **最后更新**: 2026-03-30  
> **用途**: 基于真实网站的具体操作测试

---

## 使用说明

本文档提供两种测试方式：

1. **分步测试** - 按照操作步骤逐条执行，适合调试和定位问题
2. **完整流程** - 直接复制粘贴给 Copilot，让 AI 自动完成整个任务

**推荐用法**：先尝试完整流程，如果卡住再切换到分步调试。

### 目标

## 场景 13b: 右键保存图片（另存为）

### 目标
测试 saveImage 操作 - 右键点击嵌入式图片并保存（无需下载链接）

### 前置条件
- 使用 Unsplash 或其他有嵌入式图片的网站

### 完整流程（直接复制给 Copilot）

```
打开 Unsplash 首页 https://unsplash.com，等待页面加载完成后，找到页面上的第一张图片，使用 saveImage 操作在图片位置右键保存，保存完成后告诉我图片的文件名、大小和保存路径。
```

### 分步操作

| 步骤 | AI 指令 | 预期结果 |
|------|---------|----------|
| 1 | "打开 https://unsplash.com" | 页面加载，显示图片网格 |
| 2 | "截图查看第一张图片位置" | 确认图片坐标 |
| 3 | "使用 saveImage 操作保存第一张图片" | 在图片坐标处右键并保存 |
| 4 | "告诉我图片保存信息" | AI 告知文件名、大小和完整路径 |

### AI 操作示例

```json
// 基础用法：自动生成文件名（推荐）
{"action": "saveImage", "target": {"type": "coordinate", "x": 0.25, "y": 0.4}, "description": "右键保存页面第一张图片"}

// 指定文件名（系统会自动处理重复）
{"action": "saveImage", "target": {"type": "coordinate", "x": 0.25, "y": 0.4}, "savePath": "my-image.png", "description": "右键保存图片并命名为 my-image.png"}
```

### 重要说明
- **强制工作区限制**：无论 AI 是否指定 `savePath`，文件最终都会保存在 `~/.siliu/workspace/downloads/` 目录下（如果 AI 指定了外部路径，会被强制改为工作区内）
- **自动防重复**：如果文件名已存在，系统会自动添加序号，如 `my-image.png` → `my-image(1).png`
- **自动生成文件名**：如果不指定 `savePath`，系统使用 `image-{timestamp}.png` 格式

### 验证方法
保存完成后，系统会向 AI 发送消息：
```
图片 "image-xxx.png" (245 KB) 已保存完成，路径: C:\Users\...\.siliu\workspace\downloads\image-xxx.png
```

### 通过标准
